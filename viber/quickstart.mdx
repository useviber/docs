## ğŸš€ Quick Start (2 minutes)

### Step 1: Install Viber CLI

```bash
npm install -g @useviber/cli
```

<Tip>
The CLI automatically installs the MCP server. If you get permission errors, run: `viber mcp:install`
</Tip>

### Step 2: Start Viber in Your Project

Navigate to your project directory and run:

```bash
cd your-project
viber start
```

**What happens automatically:**
- Viber auto-detects your framework and ports
- Creates a proxy server that intercepts your app
- Injects the Viber widget into your HTML automatically
- Opens your browser with the proxied version
- You see the floating Viber widget (ğŸ¤–) immediately

### Step 3: Use Viber

1. **Your app opens automatically** in the browser (through Viber's proxy)
2. **Look for the floating Viber widget** (ğŸ¤–) in the bottom-right corner
3. **Click the widget** to open the Viber interface
4. **Click any element** on your page to select it
5. **Describe your changes** using natural language
6. **Watch the magic happen!** âœ¨

## ğŸš¨ Troubleshooting

### "Port already in use"
```bash
# Kill the process using the port
lsof -ti:8426 | xargs kill -9

# Or use a different port
viber start --viber-port 9000
```

### "Viber not connecting"
```bash
# Check if your dev server is running
npm run dev

# Try debug mode
viber start --debug
```

### "Widget not appearing"
- **Make sure you're accessing the proxy URL** (http://localhost:8426), not your original dev server
- Refresh the page
- Check browser console for errors
- Ensure Viber CLI is running

### "Framework not detected"
```bash
# Force framework detection
viber start --framework react

# Check project structure
ls -la package.json
```

## ğŸ” What's Actually Happening

### Behind the Scenes

When you run `viber start`:

1. **Port Detection**: Viber scans for your running dev server
2. **Proxy Creation**: Creates a proxy server on port 8426
3. **Plugin Injection**: Automatically injects Viber scripts into your HTML
4. **Widget Creation**: Adds the floating Viber widget to your page
5. **Communication Setup**: Establishes WebSocket connections for real-time updates
